# FAILSAFE: AI-Powered Early Intervention System

FAILSAFE is a full-stack early intervention platform that uses XGBoost, FastAPI, and React to predict student failure risk before it is too late. It combines explainable machine learning with a modern web interface to generate SHAP-backed risk insights and actionable intervention plans for faculty.

## Problem Statement

In educational institutions, student failure often goes undetected until end-of-semester results, leaving no room for meaningful intervention. Faculty lack a proactive, data-driven tool to identify at-risk students early and understand the root causes behind their struggles. To address this, FAILSAFE provides a web-based system where faculty upload student data. A trained ML model predicts failure risk?with Explainable AI making every prediction transparent?and auto-generates personalized intervention plans to help faculty act before it's too late.

## Deliverables & Goals Achieved

- Predict at-risk students early using attendance, assignments, and behavioral data.
- Use SHAP (SHapley Additive exPlanations) to explain why each student is flagged, making predictions trustworthy and actionable for non-technical faculty.
- Auto-generate personalized interventions per student (e.g., extra classes, counseling referrals, study plan adjustments).
- Provide a robust React dashboard for faculty to track risk trends, apply interventions, and monitor improvement over the semester.

```text
├── backend/                 # FastAPI application and API services

│   ├── services/            # ML inference service and SHAP explainers
│   ├── database.py          # Database connection setup
│   ├── main.py              # FastAPI application entry point
│   └── models.py            # SQLAlchemy database models
├── frontend/                # React application (Vite + Tailwind)
│   ├── src/
│   │   ├── components/      # Reusable UI components (AppShell, RiskProfilePanel)
│   │   ├── lib/             # API clients (Axios) and Context providers
│   │   ├── pages/           # Page views (Auth, Dashboard, Upload)
│   │   ├── App.jsx          # Routing configuration
│   │   └── main.jsx         # React DOM rendering
│   ├── package.json         # Frontend dependencies
│   └── tailwind.config.js   # Tailwind CSS configuration
├── data/                    # Local data (Ignored by Git)
│   ├── raw/                 # Raw CSV datasets (e.g., student-mat.csv)
│   └── models/              # Exported XGBoost model artifacts (.pkl)
├── scripts/                 # Independent Python scripts
│   └── train_pipeline.py    # XGBoost model training and export pipeline
├── .gitignore               # Git ignore rules (venv, node_modules, .db, data)
└── README.md                # Project documentation
```

- `data/raw/` stores the raw student CSV files used for model training and testing.
- `data/models/` stores the exported `.pkl` model artifacts generated by the training script.
- `scripts/` contains the ML pipeline used to train and export the XGBoost model.
- `backend/` contains the FastAPI application, database models, authentication logic, and ML inference service.
- `frontend/` contains the React dashboard, upload workflow, authentication UI, and charting components.

## Complete Step-by-Step Setup & Deployment Guide

### 1. Clone the Repository

```bash
git clone https://github.com/RISHI41221/failsafe.git
cd failsafe
```

### 2. Download Raw Data

Download the "Student Performance Data Set by UCI" from Kaggle. After downloading it, place these two files inside the `data/raw/` directory:

- `student-mat.csv`
- `student-por.csv`

### 3. Python Environment Setup

Create a virtual environment:

```bash
python -m venv venv
```

Activate it on Windows:

```powershell
venv\Scripts\activate
```

Activate it on Mac/Linux:

```bash
source venv/bin/activate
```

### 4. Install Backend Dependencies

```bash
pip install fastapi uvicorn sqlalchemy passlib bcrypt python-jose[cryptography] python-multipart pandas xgboost shap scikit-learn joblib
```

### 5. Train the ML Pipeline

Generate the model artifacts before booting the backend server:

```bash
python scripts/train_pipeline.py
```

This command trains the XGBoost model and saves the exported `.pkl` files into `data/models/`.

### 6. Deploy the Backend

By default, the backend is configured for PostgreSQL in `backend/database.py`. For quick local testing, you can switch to the commented SQLite fallback in that file before starting the server.

```bash
cd backend
uvicorn main:app --reload
```

The backend will run on `localhost:8000`.

### 7. Deploy the Frontend

Open a new terminal, then run:

```bash
cd frontend
npm install
npm run dev
```

The frontend will run on `localhost:5173`.

### 8. Test the System

Open your browser to `localhost:5173`, register a faculty account, navigate to the Upload page, and drop the `student-mat.csv` file to see the system in action.
